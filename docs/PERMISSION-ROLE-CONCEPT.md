# 권한·역할 개념 정리

## 1. 권한 (Permission)

**권한 = 사이트 내 존재하는 모든 버튼(액션)에 대한 실행 권한**입니다.

- 각 버튼·액션(생성, 수정, 삭제, 컨펌 요청, QA 반영 실행 등)마다 대응하는 권한 코드가 있습니다.
- 사용자가 해당 권한을 가지고 있어야 그 버튼이 노출되고, 실행 시 백엔드에서도 동일 권한으로 검사합니다.
- **역할(role)이 아니라 권한(permission)으로만** 수행 여부를 판단합니다.

예: `my_dashboard.execute_qa` → "QA 반영 실행" 버튼 실행 권한.

---

## 2. 권한 설정 (그룹화)

**권한 설정은 각 페이지(메뉴)별로 그룹화**해서 관리합니다.

- 역할 편집 화면에서 "프로덕트", "쿼리 템플릿", "나의 대시보드", "이벤트 생성" 등 **페이지 단위 그룹**으로 권한 목록이 묶여 있습니다.
- 같은 권한이 두 그룹에 중복으로 나오지 않도록, 실행 권한 등은 한 그룹에만 두었습니다.
- 그룹은 **표시/관리 편의**를 위한 것이고, 실제 동작은 여전히 개별 권한 코드 단위입니다.

참고: 그룹·권한 코드 정의는 프론트 `ARR_PERMISSION_GROUPS`(예: `front/src/types/index.ts`)와 매트릭스 문서에 정리되어 있습니다.

---

## 3. 역할 (Role) / 역할 코드

**역할 = 권한들을 묶어서 부여한 단위**입니다.

- 역할 코드(예: `admin`, `dba`, `game_manager`)는 **역할 식별자**이고,
- 각 역할에는 **여러 권한이 부여**됩니다 (예: DBA 역할 → 나의 대시보드 보기, 상세, DBA 컨펌, QA 반영 실행, LIVE 반영 실행).
- 사용자는 하나의 역할을 가지며, 로그인 시 해당 역할에 부여된 권한 목록(및 백엔드 확장 규칙)으로 메뉴·버튼 노출과 API 호출이 제어됩니다.

정리: **역할 코드 = 역할에 권한들을 부여한 묶음의 이름**입니다.
