# 테이블 기능 vs 라이브러리 정리

우리 프로젝트에서 구현한 테이블 액션과, 비슷한 기능을 제공하는 라이브러리 정리.

---

## 우리가 구현한 기능

| 기능 | 설명 |
|------|------|
| 헤더 좌측 | 리사이즈 핸들 (좌드래그 → 너비 변경) |
| 헤더 중앙 | 순서 변경 드래그 (컬럼 위치 변경) |
| 헤더 우측 | 리사이즈 핸들 (우드래그 → 너비 변경) |
| 헤더 더블클릭 | 데이터 행 셀 내용 기준 자동 너비 맞춤 |
| (미구현) 카드/행 보기 | 테이블 ↔ 카드·리스트 뷰 전환 |

---

## 한 번에 다 제공하는 라이브러리

**“헤더 좌/중앙/우 + 더블클릭 자동 맞춤 + 테이블↔카드/행 보기”를 전부 기본 제공하는 React 라이브러리는 없음.**

- 리사이즈 + 리오더는 많은 그리드/테이블 라이브러리에 있음.
- **더블클릭으로 내용물 기준 자동 맞춤**은 옵션으로 두는 곳이 적음.
- **테이블 ↔ 카드/행 보기 토글**은 “테이블 전용” 라이브러리보다는 “데이터 뷰” 쪽에 가깝고, 테이블 고급 액션(리사이즈/리오더/자동맞춤)까지 같이 제공하는 건 드묾.

---

## 기능별로 가까운 라이브러리

### 1. 컬럼 리사이즈 + 리오더(드래그)

- **TanStack Table (React Table v8)**  
  - 리사이즈·리오더 예제 있음 (headless라 UI는 직접 구현).
  - [Column Resizing](https://tanstack.com/table/v8/docs/guide/column-sizing), [Column DnD](https://tanstack.com/table/v8/docs/framework/react/examples/column-dnd).
- **AG Grid**  
  - 리사이즈·리오더·고정 기본 제공 (상용 기능 많음).
- **Mantine React Table**  
  - `enableColumnResizing`, 리사이즈 핸들 등 제공.
- **PrimeReact Table**  
  - 컬럼 리사이즈·리오더 옵션 있음.

### 2. 더블클릭/자동으로 “내용물 크기에 맞춤”

- **AG Grid**  
  - `autoSizeColumn()` / `sizeColumnsToFit()` 등으로 컬럼 너비 자동 조절 가능.
- **TanStack Table**  
  - 컬럼 사이징 API는 있으나, “셀 내용 기준 자동 맞춤”은 직접 구현해야 함.
- 대부분 그리드 라이브러리는 “한 번에 맞춤” 위주이고, “헤더 더블클릭 시 이 컬럼만 내용 기준” 같은 동작은 커스텀인 경우가 많음.

### 3. 테이블 ↔ 카드/행(리스트) 보기

- **PrimeReact DataView**  
  - **그리드 / 리스트 레이아웃 전환** 기본 지원 (`DataViewLayoutOptions`).
  - 같은 데이터를 테이블이 아닌 카드·리스트로 보여주는 용도에 가깝고, “테이블 모드에서의 컬럼 리사이즈·리오더”는 Table 쪽 설정을 봐야 함.
- **react-data-view** (stolostron)  
  - 테이블 vs 카탈로그 카드 뷰 전환에 특화. 리사이즈·리오더는 없을 가능성 높음.
- **MUI X DataGrid**  
  - `unstable_listView`로 리스트 뷰 지원 (실험적). 카드 레이아웃은 없음.

---

## 정리 및 추천

- **“지금 우리가 쓰는 방식”(Ant Design Table + 커스텀 헤더)**  
  - 헤더 좌/중앙/우 액션, 더블클릭 자동 맞춤까지 **한 라이브러리에 다 들어있진 않아서**, 지금처럼 Ant Design Table 위에 직접 구현하는 방식이 합리적임.
- **나중에 카드/행 보기까지 넣고 싶다면**  
  - **같은 데이터 소스**로  
    - 테이블: 기존 `AppTable` (헤더 액션 유지),  
    - 카드/행: 별도 컴포넌트(예: `CardList`, `RowList`)  
  - 를 두고, 상단에서 “테이블 / 카드 / 행” 토글로 스위칭하는 식이면, **TanStack Table 같은 걸로 통째로 갈 필요는 없음**.
- **전체를 한 번에 바꾸고 싶다면**  
  - **TanStack Table**로 테이블 로직을 옮기고,  
    - 헤더는 지금처럼 “좌 리사이즈 / 중앙 리오더 / 우 리사이즈” + 더블클릭 자동 맞춤을 직접 구현하고,  
    - 카드/행 보기는 같은 데이터로 별도 레이아웃 렌더링  
  - 이 조합이 유연하고, 우리가 원하는 동작에 가장 가깝다.

**결론:** 우리가 말한 “테이블 액션 + 테이블 형식 + 카드 보기/행 보기”를 **한 라이브러리가 전부 구현해 주진 않음**. 지금 구조 유지하고, 카드/행 보기만 추가하는 방식이 가장 현실적이다.
